What is virtBox?

virtBox is a cloud storage provider for online learners. The goal of virtBox is to increase productivity with an organized and clean user interface while streamlining the user flow process for saving contents online.

Design Roles

  • User Research
  • UX/UI Design
  • Brand and Identity

Tools

  • Figma
  • InVision
  • Usability Hub
  • Balsamiq
  • Draw.io

OVERVIEW

There are many strong competitors in the market such as Google Drive, Box, and Dropbox, but there is still room for the right combination of features to meet our audience needs. Here are some problems identified:

  • Problem: Many cloud storage providers lack UI simplicity

  • Solution: Clean user interface for simplicity

  • Problem: Difficult to locate file types and little support to save links

  • Solution: Organized file types with support to save links

  • Problem: Little support for real-time collaboration

  • Solution: In-house messaging system and identification

COMPETITIVE ANALYSIS

Google Drive is the most popular due to its accessibility. There are sub-environments for each file types which can be accessed anywhere and share with anyone in real-time collaboration.

Dropbox is another strong competitor. Their dashboard navigation changes based on what the users clicked. This caused confusion. It is a platform known for syncing files from multiple devices rather than for creating new contents.

Box, although not as popular, it has the fastest growing speed in the market. It targets businesses and corporates. Box is known for its collaboration features and its support for MS Office (which is highly desired based on the survey results).

What can I do with this information?

Google Drive, Dropbox, and Box all known for a specific feature. So I thought, “What should virtBox be known for?”. After careful considerations of virtBox’s target audience and goals to help online learners, I decided it should be known for its organization.

  • Google Drive: accessibility
  • Dopbox: synchronization
  • box: collaboration
  • virtBox: organization
  • Sounds about right.

PHASE 3: VISUAL DESIGN

WIREFRAMES: TURNING IDEAS INTO DESIGNS

I created paper wireframes to connect the concepts and user flows, then digitized them using Balsamiq. The differentiator of virtBox is its organization feature, which I found an opportunity to highlight in the dashboard navigation pane. This section helps the user easily locate their files based on the file type.

USABILITY TESTS

The wireframes were then used to performed usability tests: 1 in-person, and 2 remotes. This helped me see how users interact with the sketches without visual clues.

Users were asked to perform the followings:

  • Signing up for an account
  • Adding a piece of content
  • Organizing a piece of content

I found that:

  • Buttons need to be more obvious
  • There was confusion whether or not a piece of content was successfully added
  • Users could not exit popup windows
  • No differentiation of file types
  • A user commented she would use virtBox because she could easily find her files with the dashboard navigation pane!
Takeaways

The dashboard navigation pane layout is heading in the right direction.

LOGO SKETCHES

The process started off with a brainstorming session played off of the mind-mapping/keywords that represent the goal of the product. Then I narrowed down to the final five sketches. These five sketches were then further narrowed down to one logo to derive into more digitized versions.

BRANDING IDENTITY

Since the product is created for online learners, I want to establish a bright, clean, and motivative environment. The name virtBox is the combination of academic growth and the concept of online storage. Virt is short for virtual, representing the virtual workspace for users. Virt in French represents the color green and green represent growth. When users use virtBox for their online learning experience, they are growing academically. Box represents a platform to store all files in one place. virtBox’s logo combines the letter”B” sideways inside a laptop. The internal “B” is a derivative of earlier sketches of a virtual reality glasses.

COLOR & STYLE GUIDE

During my research, I realized all three of my competitors used blue as their primary color. Further research shows blue is the most widely liked color. I decided to use blue, but also wanted to differentiate my product from the competitors, thus, indigo was chosen. Indigo was paired with a blue-gray color at different opacity to create variety. All combinations passed the WCAG AA and WCAG AAA test with accessibility for those with color blindness.

PHASE 4: TESTINGS

USER PREFERENCE TESTS

I picked 3 elements that potentially need improvement. I created 2 versions for each element and conduct a preference test using Usability Hub.

2ND ITERATION

The 2nd iteration entails improvements from the feedback I received from the preference tests.

  • Dashboard - File types differentiated with icons
  • Dashboard - Collaboration feature highlighted with visual design

3RD ITERATION

Round 3 of iteration focus on detailed improvements for the homepage.

USABILITY TEST ROUND 2

The 3rd iteration then used to perform another usability tests: 2 in-person and 1 remote test. Overall, participants did not have trouble sorting content and identifying file types. Major improvements need to be fixed are with Invision. Many screens were not displaying correctly. Otherwise, participants liked virtBox features; especially the color-coded feature. They were also excited to have a semi-final product that they can experience compared to the clickable wireframes.

Things to improve:

  • Popup windows need to pop up correctly. Blur out the background.
  • Consistent sort by content format.
  • “Create new file” options need to link to the correct destinations.

FINISHING TOUCHES: A USER PERSPECTIVE

This iteration utilized the semi-final product with improvements made based on the feedback from the usability tests. Most edits were made by adding colors to sections to make information stand out and easy to read. Others were spell check and alignment check.

WHAT I LEARNED

This project was challenging for me. I enjoyed the visual design process, however, I had a hard time differentiating my product from the competitors. The current market is saturated and I did not want to create another replication of what’s already on the market. Competing with a strong competitor like Google Drive was tough. However, I learned to create the product specifically for a targeted audience. This significantly helped me plan out my project. If given more time, I would do a preference test and usability test after every iteration. I would also perform the tests with more random participants instead of distributing them to mostly the design community. I feel that this cause the results to be skewed.
